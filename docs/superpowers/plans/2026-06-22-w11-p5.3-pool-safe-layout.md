# W11 P5.3 â€” Multi-pool generation + pool-safe auto-layout (plan)

> **For agentic workers:** implement task-by-task. Steps use `- [ ]` checkboxes.

**Goal:** AI generation produces true multi-pool collaborations (stacked
horizontal pools + dashed message flows) that render in bpmn-js, while
single-party processes keep today's exact single-process behavior.

**Architecture:** A new headless `layoutCollaboration(xml)` in `@claril/bpmn-parse`
wraps the upstream `bpmn-auto-layout` (which can't lay out collaborations): it
lays out each pool's process via `layoutProcess`, harvests the DI, stacks pools
vertically with left name-bands, and synthesizes message-flow edges. The
generator prompt is relaxed to emit collaborations only for independent parties.
`generateDiagramFromPrompt` calls `layoutCollaboration` instead of `layoutProcess`.

**Tech Stack:** TypeScript, `bpmn-moddle`, `bpmn-auto-layout`, vitest.

### Task 2: `layoutCollaboration` implementation

**Files:** Create `packages/bpmn-parse/src/layout-collaboration.ts`

Behavior:
- Parse `xml` with `bpmn-moddle`.
- Find the `bpmn:Collaboration` rootElement and its participants. If there is
  no collaboration, or `participants.length <= 1`, return `await layoutProcess(xml)`.
- For each participant with a `processRef`:
  - Build a standalone `<definitions>` containing only that process and run it
    through `layoutProcess`; reparse the result and read
    `definitions.diagrams[0].plane.planeElement` (the `bpmndi:BPMNShape` /
    `bpmndi:BPMNEdge` list).
  - Compute the bbox of that process's shapes; translate all of its shapes
    (bounds + label bounds) and edges (waypoints + label bounds) by
    `(LABEL_BAND + PAD_X, cursorY + PAD_Y)`.
  - Create a `bpmndi:BPMNShape` for the participant (`isHorizontal: true`) whose
    bounds enclose the translated content + left name-band + padding.
  - Advance `cursorY` by `poolHeight + POOL_GAP`.
- Build one `bpmndi:BPMNDiagram`/`bpmndi:BPMNPlane` (`plane.bpmnElement =
  collaboration`) holding every participant shape + every translated shape/edge.
- For each `bpmn:MessageFlow`, resolve the source/target shapes (by element id)
  and synthesize a `bpmndi:BPMNEdge` with orthogonal vertical waypoints (exit the
  side of the upper element facing the lower, enter the lower's facing side).
- Set `definitions.diagrams = [diagram]` and return `(await moddle.toXML(definitions, { format: true })).xml`.

Constants: `LABEL_BAND = 30`, `PAD_X = 30`, `PAD_Y = 30`, `POOL_GAP = 60`.
